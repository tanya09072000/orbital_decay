Инструмент для численного прогнозирования эволюции орбиты низколетящего
космического аппарата под действием аэродинамического торможения и
гравитационного возмущения J₂ (сплюснутость Земли).

## Назначение

Проект предназначен для оценки **снижения большой полуоси орбиты (SMA)**
низкоорбитального космического аппарата за заданный промежуток времени.
Основная задача — получить временной ряд `SMA(t)` с учётом реальной
солнечной активности и атмосферной плотности.

Типичная область применения:

- расчёт остаточного ресурса орбиты (срок жизни до входа в атмосферу);
- оценка потребности в коррекции орбиты;
- анализ чувствительности к баллистическому коэффициенту и солнечной активности.

---

## Что рассчитывает программа

На каждом витке программа:

1. Вычисляет текущий **орбитальный период** из vis-viva уравнения.
2. Запрашивает **плотность атмосферы** для текущей высоты и даты по модели CIRA-86.
3. Интегрирует уравнение движения методом **ODE89** до ближайшего
   **восходящего узла** (событие в интеграторе).
4. Фиксирует значение **большой полуоси** и **высоты** в точке узла.

На выходе формируется:

- График `SMA(t)` — снижение большой полуоси во времени.
- Таблица с датой, SMA и высотой на каждом витке.
- Консольный вывод: итоговое снижение SMA, начальное и конечное значения.

## Физические модели

| Эффект | Модель |
|--------|--------|
| Центральное притяжение | Закон всемирного тяготения (Ньютон) |
| Несферичность Земли | Зональная гармоника J₂ |
| Аэродинамическое торможение | Квадратичный закон лобового сопротивления |
| Вращение атмосферы | Со-вращение с Землёй (ω⊕ = 7.292·10⁻⁵ рад/с) |
| Плотность атмосферы | **CIRA-86** (Committee on Space Research) |
| Солнечная активность | F10.7 и Kp из таблицы прогноза/наблюдений |
| Численное интегрирование | ODE89 (Рунге–Кутта–Вернер 8(9) порядка) |
| Интегрирование CIRA | Метод Boole по логарифмической сетке высот |

Система координат — **ECI** (Earth-Centred Inertial, J2000).

## Входные данные

Все параметры задаются в функции `setup_params` внутри `altitude_VU.m`.

### Начальные орбитальные элементы

```matlab
params.input_data = [a, e, i, Omega, omega, nu]
```

| Параметр | Обозначение | Единицы | Пример |
|----------|-------------|---------|--------|
| Большая полуось | `a` | км | `6908.889` |
| Эксцентриситет | `e` | — | `0.001009` |
| Наклонение | `i` | градусы | `97.401` |
| Долгота восх. узла | `Ω` | градусы | `317.904` |
| Аргумент перигея | `ω` | градусы | `357.882` |
| Истинная аномалия | `ν` | градусы | `2.118` |

### Параметры КА

```matlab
params.m_dry         = 306.4;   % масса без топлива, кг
params.m_fuel        = 3.6;     % масса топлива, кг
params.m_total       = 310.0;   % стартовая масса, кг
params.Cx            = 3.56;    % коэффициент лобового сопротивления
params.Sm            = 2.25;    % площадь миделя, м²
```

### Параметры расчёта

```matlab
params.duration_days = 28.9;                          % длительность, сут
params.start_dt      = datetime(2025, 8, 31, 23, 18, 47); % момент старта
params.data_file     = 'Koeff_future.csv';            % файл солнечных данных
```

Для расчёта другого КА достаточно изменить значения в `setup_params`.

## Описание файлов

### `altitude_VU.m` — точка входа

Главная функция, разбитая на четыре внутренние подфункции:

| Подфункция | Задача |
|------------|--------|
| `setup_params()` | Возвращает структуру с параметрами КА и расчёта |
| `load_solar_data(filename)` | Загружает и разбирает `Koeff_future.csv` |
| `run_simulation(params, data)` | Основной цикл по виткам, интегрирование ОДУ |
| `plot_results(results, days)` | Построение графика SMA(t) |
| `print_summary(results)` | Консольный вывод итогов и таблицы |

---

### `two_body.m` — уравнение движения

```
dy/dt = f(t, y, Cx, Sm, m_total, rho)
```

Правая часть ОДУ, складывает три ускорения:

- **`a_grav`** — центральное гравитационное притяжение `−μ r / |r|³`
- **`a_J2`** — поправка на сплюснутость Земли (пропорциональна `Re²/|r|⁵`)
- **`a_drag`** — аэродинамическое торможение: `−½ Cx Sm ρ |v_rel| v_rel / m`

Относительная скорость `v_rel` учитывает вращение атмосферы.

---

### `vector_sostoyania.m` — элементы → вектор состояния

Преобразует кеплеровы орбитальные элементы `[a, e, i, Ω, ω, ν]`
в декартов вектор состояния `[x, y, z, Vx, Vy, Vz]`.

Алгоритм: вычисление через фокальный параметр `p = a(1−e²)`, орбитальную
скорость в перифокальной системе и три последовательных поворота (Ω, i, ω).

---

### `element.m` — вектор состояния → элементы

Обратное преобразование: `[x, y, z, Vx, Vy, Vz]` → `[a, e, i, Ω, ω, ν]`.

Использует вычисление через **орбитальный момент импульса** h и
**вектор Лапласа–Рунге–Ленца** e⃗. Реализованы все шесть классических
элементов, включая обработку особых случаев (экваториальная и круговая
орбиты).

---

### `cira.m` — модель атмосферы CIRA-86

Реализует **COSPAR International Reference Atmosphere 1986**.
Вычисляет температуру, плотность и концентрации шести компонент
(`N₂, O₂, O, Ar, He, H`) для высот от 90 км и выше.

Алгоритм разбит на три высотных слоя:

| Слой | Диапазон | Шаг log(z) |
|------|----------|-----------|
| Нижний | 90–100 км | `R1 = 0.01` |
| Средний | 100–500 км | `R2 = 0.025` |
| Верхний | > 500 км | `R3 = 0.075` |

В каждом слое применяется **метод Boole** (5-точечные веса Ньютона–Котеса)
по логарифмической сетке — вынесен в локальную подфункцию `boole_integrate`.

---

### `cira_main.m` — обёртка CIRA

Готовит входные параметры для `cira.m`:

- извлекает из таблицы реальные значения `F10.7` и `Kp` по дате;
- вычисляет **склонение Солнца** по приближённой формуле (точность ±1°);
- возвращает `[TEMP, AMMW, RHO]`.

```matlab
[TEMP, AMMW, RHO] = cira_main(alt_km, date, data)
```

---

### `orb_constants.m` — физические константы

Единый источник констант, используемых во всём проекте:

```matlab
c = orb_constants()
```

| Поле | Значение | Единицы |
|------|----------|---------|
| `c.mu` | 398600.4415 | км³/с² |
| `c.Re` | 6378.137 | км |
| `c.omega` | 7.2921150·10⁻⁵ | рад/с |
| `c.J2` | 1.08262668·10⁻³ | — |

---

### `tlocal.m` — температура по высоте

Кусочная аналитическая модель температуры:

- при `z ≤ 125 км` — полином по `(z − 125)`;
- при `z > 125 км` — функция арктангенса, асимптотически стремящаяся к `T_inf`.

---

### `grav.m` — ускорение свободного падения

```matlab
g(z) = g₀ / (1 + z / R_polar)²
```

Где `R_polar = 6356.766 км` — полярный радиус Земли.

---

### `ambar.m` — средняя молекулярная масса

Полином седьмого порядка по `(z − 90 км)`, аппроксимирующий изменение
состава атмосферы от ~28.83 г/моль на 90 км до ~27.6 г/моль на 100 км.

---

### `ascendingNodeEvent.m` — событие ODE

Используется как `Events` в `odeset`. Фиксирует момент, когда
z-компонента вектора положения меняет знак с отрицательного на
положительный (пересечение экваториальной плоскости снизу вверх).

```matlab
value     = y(3);    % z = 0 → экватор
isterminal = 1;      % остановить интегрирование
direction  = +1;     % только восходящее пересечение
```

## Формат файла солнечных данных

`Koeff_future.csv` — ежедневные данные солнечной и геомагнитной активности.
Используемые столбцы:

| Столбец | Описание |
|---------|----------|
| `DATE` | Дата в формате `yyyy-MM-dd` |
| `F10.7_OBS` | Наблюдённый поток радиоизлучения Солнца на 10.7 см, s.f.u. |
| `F10.7_OBS_LAST81` | 81-суточное скользящее среднее F10.7 |
| `KP_SUM` | Сумма 8 трёхчасовых значений Kp за сутки (в единицах 0.1 Kp) |

В `cira_main.m` вычисляется средний Kp:

```
Kp_mean = KP_SUM / 80
```

Файл покрывает период с **2020-01-01** по **2041-10-01** (наблюдения + прогноз).
Строки с типом `PRD`/`PRM` в столбце `F10.7_DATA_TYPE` — прогнозные значения.
